Running Samples — Individual & Series

How a run works in PeakSimple — single injections off the gas stream and automated series runs.

2Methods & Control Files

In PeakSimple, a "method" is saved as a control file (a .con file). This single file is the heart of reproducible analysis: it bundles every setting for a run so that you — and the next student — get identical conditions every time.

What's inside a control file

A .con file packages all of these together, so loading one restores the whole method:

PieceWhat it stores
.TEMTemperature program (oven ramps, detector/valve temps)
.EVTEvent table — the timed relay actions, e.g. when the sampling valve injects
.CPTComponent table — your named peaks + retention windows + their calibration curves
.CALCalibration curves for each component
.GRAGraph/display settings
.CHRThe chromatogram data itself (one per channel/run)

Load an existing method

  1. Open your method: File > Open control file, then choose your lab's .con (e.g. RWGS.con). This loads temperatures, the event table, named peaks, and calibrations all at once.
  2. Check the temperatures took. The status zones should now show your method's setpoints. If the GC is warming up, give it time to reach them.

Modify the temperature program

The oven temperature program controls separation. To view or change it:

  1. Right-click anywhere on the chromatogram → Temperature. The Temperature control window opens.
  2. Click Add to open Temperature segment details. Each segment has three numbers: Initial temperature, Hold for (minutes), and Then ramp at (°C/min) Until temperature. Example: start 50 °C, hold 3 min, then ramp 20 °C/min until 250 °C.
  3. Click OK. Add more segments for multi-ramp programs. Click OK to close.
⚠Always save your changes back into the method After editing temperatures, the event table, or components, save with File > Save control file. If you don't, your change is lost the next time the method loads — and the next person gets different conditions. If you're experimenting, Save As a new name (e.g. RWGS_test.con) instead of overwriting the lab's master method.
ℹWhy this matters for RWGS Run-to-run comparisons (conversion, selectivity) are only valid if conditions are identical. Using one locked-down control file is how you guarantee that. Treat the master .con as read-only unless your PI says otherwise.

Retention windows are shown in minutes (what PeakSimple displays) and in the raw file units (minutes × 1000). Confirm each listed peak has a current .cal file on the GC PC, or it will not be quantified.

ℹWhy two CO₂ peaks, and what is "mix"? This method uses two valves: valve G injects the sample, and valve F switches the column plumbing partway through the run (~5.5 min). A gas can then reach the detector by two routes and appear twice — an early peak (e.g. "CO₂ first", ~6.7–7.4 min) from the main column path and a later peak ("CO₂ second", ~15–16 min) from the second / back-flushed path. The "mix" peak is a composite of gases that co-elute (are not separated) on one column path; it is labelled mainly to track the valve step rather than quantified on its own. Which peak is used for each gas is fixed by its retention window — confirm with a known standard which CO₂ peak your lab reports.

⚠Maintenance checks Verify these periodically on the GC PC:
  • Calibration files present. Each named peak references a .cal file by path. Confirm every component above has a current .cal in the PeakSimple folder, or that peak will not be quantified.
  • Oven temperature agrees. The control file and the temperature file should specify the same isothermal setpoint (they have differed — 45 °C vs 75 °C). Confirm the value in use and save.
  • Saved-data labels are correct. Chromatogram metadata can still carry PeakSimple defaults (e.g. HELIUM AT 5 PSI, RESTEK 15METER MXT-1, Syringe Injection) that do not match this GC's H₂ carrier, packed columns, or gas-valve sampling. Update them in Edit > Overall and channel details so saved files describe themselves correctly.
  • One calibration per gas. Keep a single current .cal per component and archive older copies so the wrong curve can't be loaded.
  • Component numbering. Make sure no two components share the same peak number.

4Run One Sample (off the gas stream)

Because you sample from a flowing gas stream with a gas-sampling valve, you don't inject by hand. The valve fills a fixed loop, then — at a precise moment during the run — rotates to push that loop onto the column. PeakSimple controls that rotation with a timed event.

Set the injection event (usually already in your method)

Your .con file should already contain this, but here's how to check or build it:

  1. Open the event table: right-click the chromatogram → Events (or Edit > Channels → Events for channel 1).
  2. Click Add. In Event details, select the relay letter assigned to your sampling valve (check the label on the right side panel of the GC — see Your Lab's Settings). Tick the On box (ON = inject).
  3. Set the Event time to when injection should happen — SRI recommends no earlier than 0.10 min (6 s) so the run can initialize.
  4. Add a second event later in the run to turn the relay OFF (untick "On") so the valve returns to LOAD and the next loop fills.
  5. Optionally add a Zero event at time 0 to auto-zero the detector baseline at the start of the run.
ℹReset relays at end of run By default PeakSimple returns all relays to OFF when a run ends (Edit > Overall → "Reset relays at end of run"). That's usually what you want so the loop is refilling between runs.

Start the run

  1. Confirm the baseline is flat and the GC is at temperature.
  2. Make sure the loop is loading: the valve relay should be OFF and your sample gas flowing through the loop.
  3. Name the run / set where it saves (so you can find the data later) per your lab's convention.
  4. Press the Run button (or your lab's start key). The clock starts at 0:00, the event table fires the valve at your set time, and peaks appear on all three channels (FID, methanizer, TCD).
  5. Wait for the run to finish (the full run time). The baseline returns and the chromatogram is complete and saved.
✓Test the valve by hand To check the valve actuates, open View > Relay/pump window and click the relay letter to toggle it (it highlights when ON). You can also hold Ctrl + the relay letter. You should hear/see the valve switch. Return it to OFF (LOAD) before a real run.

5Run a Sequence Automatically (AutoQueue)

The Autosampler Queue runs a list of control files back-to-back without you standing there. For an RWGS experiment this is how you take repeated injections off your gas stream over hours — e.g. inject every few minutes to watch conversion change over time — and how you front-load calibration standards before the real samples.

⚠SRI's own caution The AutoQueue is the most powerful and most complex part of PeakSimple. Get comfortable running single samples (Sections 6–8) first. Don't build a long unattended queue until you've done several manual runs successfully.

The idea: one control file per run

Each line in the queue is a .con control file. They must each have a unique name and live in the PeakSimple application directory. Numbering them keeps order obvious: Vial01.con, Vial02.con, …

  1. Open the queue: View > Autosampler.
  2. Add your master method: click Add…, browse to your .con file, and place it in the list.
  3. Make many copies at once: highlight your master and click Add copies. Enter how many runs you want and tick "Increment data filename for each copy" so each injection saves to its own data file (no overwriting). This is the easy way to set up, say, 30 repeat injections.
  4. Fine-tune in the spreadsheet: click Edit all to open a table of every control file's parameters. Edit a single cell, or highlight a column and use Fill to change them all. Click Update control files to save changes.
  5. Save the queue with Save… as a .QUE file so you can reload the whole sequence later with Load….
  6. Start the queue. PeakSimple runs each control file in order, saving a separate chromatogram for each.

Put calibration standards at the front

A common pattern: make the first few runs your calibration standards so the curve is fresh, then let the samples follow.

  1. Open the first control file: File > Open control file.
  2. For each channel, open Post-run actions and tick "Recalibrate at level ___", entering 1 for your Level-1 standard. File > Save control file.
  3. Repeat for the second standard with level 2, third with level 3, etc. (You can also do this from the Edit all spreadsheet.) The remaining sample runs leave recalibration off.
ℹYou must calibrate before queuing The AutoQueue uses calibration curves you've already built (Section 8). Recalibrate-at-level just refreshes those existing curves with fresh standard runs at the start — it doesn't create them from nothing.

6File Organization & Naming

Right now every run saves loose in the PeakSimple program folder — impossible to search, easy to overwrite, easy to lose. This convention ties every file to a student, a reaction, and a date. Agree on it once; everyone follows it.

Folder structure

Choose a data root outside the PeakSimple program folder, then nest: student → reaction + date → files.

D:\GC Data\                              ← data root (NOT the PeakSimple program folder)
└─ Porosoff_M\                          ← student:  LastName_FirstInitial
   └─ RWGS_2026-06-08_Pt_450C\          ← reaction _ date _ short descriptor
      ├─ MP_RWGS_Pt_450C_2026-06-08_run01.chr
      ├─ MP_RWGS_Pt_450C_2026-06-08_run01.asc
      ├─ MP_RWGS_Pt_450C_2026-06-08_calcheck.chr
      ├─ results_log.txt
      └─ README.txt        (catalyst, mass, flows, setpoints, notes)

File-name pattern

initials_reaction_catalyst_temp_YYYY-MM-DD_run##

Example: MP_RWGS_Pt_450C_2026-06-08_run01.chr  →  Marc Porosoff, RWGS over Pt at 450 °C, run 1 of that session.

The rules

RuleWhy
ISO dates — YYYY-MM-DDFiles and folders sort in true date order automatically.
No spaces — use underscores _Spaces and stray dots (e.g. run 5..ASC) break scripts and look messy.
Zero-pad run numbers — run01, run02…They sort correctly and line up with PeakSimple's auto-increment.
One folder per sessionNever mix reactions or dates in a single folder.
Short but meaningfulWho / what / conditions / when / which run — nothing else.

Make PeakSimple actually save there

  1. Create the folders first in Windows Explorer — the student folder and the reaction_date session folder — before you run.
  2. Set the save location: Edit > Overall → fill in Default data file path with that session folder. (This is the Overall controls screen shown in Section 3 — note the "Default data file path" field.)
  3. Set the data-file base name for each channel to your pattern ending in run01. With auto-increment (already on in your method) PeakSimple saves run01, then run02, run03… automatically.
  4. Point the results log (Postrun actions) into the same folder, e.g. results_log.txt.
  5. Save the method with File > Save control file so the path and naming stick.
✎Your lab's choices Data root (off the program drive): D:\GC Data\ · Student folder format: LastName_FirstInitial · Example session: RWGS_YYYY-MM-DD_catalyst_temp
⚠Two PeakSimple gotchas
  • Control files must stay put. Any .con used by the AutoQueue has to live in the PeakSimple application directory — it can't move into a student folder. Keep methods there but named clearly (e.g. RWGS_master.con); only the data goes in the organized folders.
  • Don't move calibration files after linking. .cal and .cpt files are referenced by path inside the method. Pick one permanent location and leave them there, or the links break (see the maintenance checks in The RWGS Method at a Glance).
✓Cleaning up the current mess Build the folder tree first, then copy (don't move/delete yet) existing loose .chr/.asc files into the right student/date folders. Once you've confirmed the copies, drop the originals into an Archive_pre-2026\ folder rather than deleting. Apply the naming convention to new runs going forward — you don't have to rename history.

7Quick-Start Cheat Sheet

For routine runs once you know the basics. Print this and tape it by the GC — use the 🖨 button in the sidebar to print just this page.

1 · Startup (≈20 min)

  1. H₂ ON (carrier + TCD ref + FID fuel) first; confirm flow.
  2. Power on GC + PC.
  3. Wait for all zones at setpoint.
  4. Light FID flame; confirm (fog test).
  5. Wait for flat baseline.

2 · Open PeakSimple

  1. Launch PeakSimple.
  2. If "not functioning" → check USB / Edit>Overall (board 302, port).
  3. File>Open control file → your RWGS.con.
  4. Confirm temps loaded.

3 · Run a sample

  1. Valve OFF = loop loading from stream.
  2. Baseline flat? Name/locate save file.
  3. Press Run ▶ (clock starts at 0:00).
  4. Event table injects valve (relay ON) at set time.
  5. Let the full run finish.

4 · Integrate & read

  1. Peaks land in their retention windows → named.
  2. Off-window? Drag window, press Enter.
  3. Fix overlaps: Edit>Manual Integration → Drop.
  4. Integrate samples the SAME way as standards.

5 · Calibrate (when needed)

  1. Run standard → right-click peak → Edit [gas].
  2. Pick Level → type known amount → Accept New.
  3. Repeat per level (ascending order!).
  4. Method = curve model; Statistics = fit.
  5. Save .cal; Close.

6 · Sequence (AutoQueue)

  1. View>Autosampler.
  2. Add master .con → Add copies.
  3. Tick Increment data filename.
  4. Standards first via post-run Recalibrate at level.
  5. Save as .QUE → start.

7 · Export to Excel

  1. Results button → Copy → paste in Excel.
  2. Many runs: postrun Add to results log → Copy results log.
  3. Right-click chart → Copy picture for the image.
  4. Save into …\Student\Reaction_date\ — see naming rules.

8 · Standby / Shutdown

  1. Usually standby, not full shutdown.
  2. Keep carrier (H₂) flowing always.
  3. Drop oven to standby temp; FID flame OFF.
  4. Full shutdown: cool first, then per lab H₂ policy.

⚠ Never  ·  Always  ·  Help

  • Never heat a column without carrier (H₂) flowing. Never let H₂ accumulate — keep the area ventilated and leak-checked.
  • Always integrate samples and standards identically. Always let zones stabilize before running.
  • Stuck? Ask your PI/senior student, or SRI Tech Support (310) 214-5092. Relay/valve labels are on the GC's right side panel.
