Data Analysis, Processing & Export

Turn chromatograms into numbers: integration, calibration curves, and export to Excel.

1Peak Integration

Integration is how PeakSimple measures the area under each peak — and area is proportional to how much of that gas is present. Good integration = a clean baseline drawn under each peak and the right peaks named. This is where beginners spend the most time, so go slowly.

Step 1 — Name your peaks (the component table)

PeakSimple only reports peaks you've named. For each gas you care about:

  1. Right-click on the peak → Add Component. A horizontal retention window bar appears, centered on the peak.
  2. Adjust the window: drag the small uprights at each end to make it wider/narrower; drag the center of the bar to move it. The window should comfortably bracket the peak.
  3. Double-click under the peak (or right-click → Edit Component) to open Component details. Enter a Peak number, a Peak name (e.g. CO), and a Unit (e.g. ppm or %).
  4. Choose "Show Largest Peak Only" so a stray noise blip in the window isn't mistaken for your gas. Click OK — the peak is now labeled.
  5. Repeat for every gas on that channel, then right-click → Components → Save to store the component table.

Step 2 — Check the automatic integration

PeakSimple integrates automatically using a few parameters. Right-click the chromatogram → Integration to adjust them. The most useful is Area Reject — raise it to ignore tiny noise peaks, lower it (e.g. to 1.0) to catch small real peaks. After changing anything, press Enter (or Edit > Reintegrate) to redraw.

Step 3 — Fix the baseline by hand (when needed)

Open the manual integration toolbar with Edit > Manual Integration. Click a tool, then click the valley/point you want to fix. The ones you'll actually use:

ToolUse it to…
DropSplit two overlapping peaks with a vertical line down to the baseline (most common fix)
BasedRaise the baseline up to the valley between two peaks
NoneMake the baseline ignore a small peak (its area merges into the neighbor)
Lead / Trail skimSkim a small peak off the leading/trailing edge of a big one (a shoulder)
Rubber BandDraw a custom baseline by hand between two points
InhibitStop integrating after a point (ignore everything after)
UndoReverse your manual changes
⚠Be consistent However you integrate your standards is exactly how you must integrate your samples. Inconsistent baselines between calibration and samples is the #1 source of bad numbers. Save your integration choices into the control file.

2Calibration & Calibration Curves

A peak area is just a number until you tell PeakSimple what a known amount of that gas looks like. Calibration means running standards of known concentration and letting PeakSimple build a calibration curve (area vs. concentration). After that, every unknown peak area is automatically converted into a real concentration.

Single-point calibration (the simplest case)

  1. Run (or open) a standard of known concentration that's already named in your component table.
  2. Right-click the peak → Edit [gas name] (e.g. "Edit CO"). If asked whether to copy an existing curve, click No for a fresh one.
  3. In the Recalibration Level window leave Level 1 selected → OK. The blank calibration curve opens.
  4. Type the known amount injected into the top-left cell, then click Accept New. PeakSimple auto-fills the measured peak area and draws the curve.
  5. Click Save, name it (e.g. CO.cal), then Close. (Once named, it auto-saves when you exit.)

Multi-level calibration curve (recommended for RWGS)

Several standards spanning your expected range give a far more trustworthy curve. Repeat the run-and-enter step at each level:

  1. Run/open the Level 1 (lowest) standard. Right-click the peak → Edit → select Level 1 → enter its amount → Accept New.
  2. Run/open the Level 2 standard. Right-click → Edit → select Level 2 → enter its amount in the second row → Accept New. The curve now has two points.
  3. Repeat for Levels 3–7 as needed (up to 7 levels).
  4. With ≥2 levels, click the Method button to pick the curve model: Multiple Line Segments (default), Single line through origin (Ax), or Quadratic through origin. For gas standards, a single line through origin is often appropriate.
  5. Click Statistics to see the fit quality (how well the line matches your points). Then Close.
⚠Levels must be in order You must enter calibration levels in ascending (or descending) order — both the concentration and the area must increase together. If a higher level has a smaller area (or vice-versa), PeakSimple will refuse the point. That usually means a bad run or a mis-typed concentration — re-run it rather than forcing it.
✓Calibrate every gas, on every detector Build a curve for each component you expect (CO₂, CO, CH₄, C₂H₆, C₂H₄, H₂, …) on the detector you'll quantify it with. Calibrations are stored in the component table (.CPT) and travel with your control file.

3Save & Export Data to Excel

Each run's chromatogram is saved automatically as a .CHR file (one per channel). To get the numbers — retention times, areas, and calibrated concentrations — out for analysis, copy them into Excel.

One run's results → Excel

  1. Click the Results button in the toolbar (or View > Results) to open the Results window. It lists each peak: number, name, retention time, area, calibrated result, and units.
  2. Click Copy to put the results table on the clipboard.
  3. Open Excel and Paste. Rows and columns fill in automatically.
  4. To paste the picture of the chromatogram too: right-click the chromatogram → Copy picture, then paste into Excel/Word/PowerPoint.

Many runs → one growing table (best for sequences)

For a sequence of injections you want all results stacked in one place automatically:

  1. In the Results window, click Clear results log once at the start (confirm Yes).
  2. Right-click the chromatogram → Postrun and tick "Add to results log". Now every finished run appends its results to the log automatically.
  3. When done, open the Results window and click Copy results log (note: this is a different button from plain "Copy"). Paste into Excel to get every run in one table. Show results log opens it in Notepad if you prefer.

A whole experiment at once (e.g. 25 injections over 12 hours)

You do not integrate 25 runs by hand. There are two ways to get one combined table:

Automatically, while the experiment runs

This method is already set up for it. When each injection's run ends, its post-run actions fire: PeakSimple integrates the chromatogram against the component table, saves it with an auto-incrementing filename, and appends a row to the results log. So after a 12-hour run you have 25 saved chromatograms and a results log already holding all 25 rows. Open Results → Copy results log and paste into Excel to get the whole experiment in one table.

After the fact (re-processing saved files)

If you need to re-integrate existing data — e.g. after fixing a calibration — use batch reprocessing: View > Autosampler, tick Batch reprocessing mode, click Add multiple, choose the control file, then select all 25 .chr files. PeakSimple runs each one back through that method and (with "add to results log" on) rebuilds the combined table.

ℹOne results log per channel Because this GC records three channels, each detector keeps its own results log (FID, methanizer, TCD). Export the log for each detector you quantify on, or combine them in Excel. Tip: click Clear results log once at the very start so the table holds only this experiment's runs.
✓Name files so you can find them Use clear, dated run names (and the "increment data filename" option in queues) so you can match each chromatogram back to its reactor conditions later. Save into your lab's data folder — see Your Lab's Settings.
